                        # 创建一个有范围的公共包

                        1. 如果您正在使用npmrc来管理多个注册表上的帐户，请在命令行上切换到适当的配置文件：
                        npmrc <profile-name>
                        1. 在命令行上，为您的包创建一个目录：
                        mkdir my-test-package
                        1. 导航到包的根目录：
                        cd my-test-package
                        1. 在包根目录中，运行npm init命令并将范围传递给scope标志：
                        git init
git remote add origin git://git-remote-url
                        1. 响应提示以生成package.json文件。有关命名软件包的帮助，请参阅“软件包名称准则”。
                        • 对于组织范围的程序包，请替换my-org为您的组织名称：
                        npm init --scope=@my-org
                        • 对于用户范围的程序包，请替换my-username为您的用户名：
                        npm init --scope=@my-username
                        1. 创建一个README file ，解释您的程序包代码是什么以及如何使用它。

                        # 发布范围内的公共软件包

                        npm publish --access public
